Booth arrangement

(file name: booth.pl)

Kim has just finished setting up booths for a conference poster session. However now an extra booth has been added to the plan.

Each booth has a fixed width and depth. At each step Kim may move a booth by distance 1 in any cardinal direction (left, right, forward, backward), except it may not pass through another booth. How many steps will it take to move the new booth to its target location, preserving the positions of all other booths?

Input format

An input file for LP systems contains the following facts:

  • One fact of the form room(W,H), which specifies the width W and height H of the room (3 ≤ W, H ≤ 20).
  • One fact booths(N), which specifies the number of booths (1 ≤ N ≤ 20).
  • A relation that consists of facts of the form dimension(B, W, H), which specifies the width W and height H of booth B.
  • A relation consisting of facts of the form position(B, X, Y), specifying the initial position (X, Y) of booth B.
  • One fact target(B, X, Y), specifying the destination (X, Y) of the target booth B.
  • An additional fact horizon(H) gives an upper bound on the number of moves to be performed.

An input file for Minizinc specifies the size of the room (W, H), the number of booths (booths), the dimensions of each booth (widths, heights), the positions (xs, ys) of each booth, and the target booth (target_booth) with its destination (target_x, target_y).

Output format

The output should contain exactly one fact of the form moves(K), where K is the minimum number of moves required to rearrange the displays.
